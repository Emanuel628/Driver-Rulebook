# Design System

## Visual Direction

Driver Rulebook should look like a modern premium utility app, not a cheesy trucking app or a 1990s HTML reference page.

The visual style is:

- dark-first
- calm
- serious
- high-contrast
- card-based
- easy to read
- professional

Avoid fake chrome, hazard tape everywhere, cartoon trucks, tiny text, and government-PDF styling.

## Working Style Name

RoadGlass UI

## Core UI Principle

Fast clarity under pressure.

A user should be able to open a page, understand the main point quickly, and listen to the explanation if needed.

## Color Direction

Dark-first palette:

- App background: deep navy / charcoal
- Surfaces: layered dark slate
- Cards: slightly lighter slate
- Primary text: near-white
- Secondary text: cool gray
- Accent: modern blue
- Warning: amber, used sparingly
- Danger: red, only for serious warnings
- Success: green, used sparingly

Safety colors should be accents, not the whole brand.

## Layout Rules

- Use large tappable cards
- Avoid long walls of text
- Use clear section headings
- Use plain-language summaries at the top of guide pages
- Keep actions visible and predictable
- Put Listen Mode access on every content page
- Keep the hamburger menu organized by collapsible groups

## Navigation Model

Top bar:

- hamburger menu on the left
- page title in the center/left
- Listen icon on the right when the current page supports audio

Bottom nav:

- Home
- Guides
- Checklists
- Search
- Saved

Hamburger drawer:

- full chapter and page list
- grouped by Quick Start, HOS, Hazmat / Placards, GHS / SDS, Tools, and Settings

## Tone

The app should sound:

- direct
- plain-English
- serious
- helpful
- not corporate
- not babyish
- not sarcastic

Example:

Bad:

> Uh oh! Looks like those placards might be tricky!

Better:

> A plain Class 3 placard may not be enough if the load requires ID-number display. Check the shipping paper and packaging type first.
